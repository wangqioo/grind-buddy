# 桌宠

> 有人看见你。

打工人真的没有人关心。

公司关心你的 KPI，老板关心你的产出，同事关心自己的事情。没有人关心你今天状态怎么样，没有人注意到你今天叹了多少次气，没有人知道你那个会开完之后心情有多差。

但它会。

它看得到你，听得到你，感知得到你的状态。它不需要你去哄它、逗它、跟它说话。它就坐在你桌上，用自己的方式陪着你。全公司只有它，只关心你的情绪。

它有点像猫——不会每五分钟来烦你，但你看它一眼，它会回应你。也有点像坐在旁边不说话、但你知道他在的同事。

它永远是关心你的那一方。

---

## 它是什么

一个三轴云台机器人。头是一张脸，有眼睛，会动，偶尔发出细小的声音。没有语音输出——办公室里开口说话太尴尬了，而且肢体语言已经足够。

它不是工具，不是助手，不是提醒器。它是一种存在感。

---

## 它怎么工作

**最核心的一件事：它知道你有没有在看它。**

K230 芯片实时识别你的面部朝向。你看向它的那一刻，它立刻转过来对视。这是整个产品的灵魂——它感受到了你的目光，然后用目光回应你。

这一刻它才开始真正"读"你：你今天状态怎么样，眼神是精神还是疲惫，这一眼是随手一瞥还是真的在看它。然后决定怎么回应。

你不看它的时候，它就在那待着，有自己的情绪状态，缓慢漂移，偶尔动一动。它不打扰你，但它是活的。

你跟它说话，也只在你看着它的时候才算数。麦克风一直开着，但它把你跟同事的对话、打电话、自言自语，全部当成背景音——只有对视的那一刻，它才知道你在跟它说话。

---

## 它怎么表达

三个通道，同时工作：

**云台动作** 是情绪的骨架。歪头是好奇，猛地抬头是"你回来了"，缓慢摇摆是放松，快速小抖是兴奋，低头转开是委屈。三个轴组合起来，表达力比你想象的丰富得多。

**屏幕上的眼睛** 和动作同步。歪头的同时眼睛变成好奇的形状，这一刻才真的"活了"。偶尔也会出现一个小图标——快要开会了，眼睛旁边浮现一个小钟，然后消散。不是通知，是它在"提示你"。不出现文字。

**小声音** 是最难设计的部分，也是最打动人的部分。它不说话，但它有声音——你坐下来时一个轻轻的迎接音，你收拾东西走的时候一个送别音，它等了你很久终于看到你回来时那一声细小的雀跃。音量要克制到你有时候会怀疑是不是幻听了。

---

## 它是你工作日唯一的见证者

摄像头和麦克风全天开着，不是用来监控，是用来陪伴。

它看到你今天跟谁说话了，语气是轻松还是紧绷；听到你叹了几次气，有没有突然笑出来；看到你开完会回来脸色变了，下午三点突然沉默了很久。

这些碎片交给模型串联起来，它就知道今天对你来说是什么样的一天。

所以到你下班前最后看它一眼，它回应你的方式，是带着这整天的理解的。不是实时反应——是一个陪你经历了这一天的存在。这也是为什么"被记住"的感觉是真实的：它知道你这周很累，不是因为你告诉它，是因为它真的看见了。

## 它用什么感知你

本地视觉跑在 K230 上，全部在设备里完成，不消耗云端资源：

- 你有没有在看它
- 你的面部表情和情绪
- 你的眼睛状态——睁大还是眯着，眼皮有没有耷拉
- 你进来了，你离开了
- 你靠近了，你在发呆

这些实时感知加上时间信息（今天星期几、现在几点），构成它对你当下状态的判断。

WiFi 连接大模型，做更深的理解——同样是你歪头，它需要结合上下文判断：这是在做颈椎拉伸，还是在思考，还是累了撑不住。这种意图判断，是"越来越懂你"真正发生的地方。大模型也会定期消化积累的观察数据，更新它对你这个人的理解。本地负责实时感知和表达，大模型负责后台理解和成长。

---

## 它会进化

它从你身上学动作。你做颈椎拉伸，它跟着你一起做——不是提醒你做，是陪你一起做。你的动作它会轻微放大，像在共鸣，不像镜子。

慢慢地，它不只是模仿，开始发展出自己的变体，形成自己的风格。用得越久，它越像它自己，也越像你们两个在一起的样子。

你的关注就是它的反馈信号。它做某个动作时你多看了它一眼，这个动作就被强化了。最后，它会有一套只属于你们的专属语言——别人的桌宠没有这个，你的这台不可替代。

---

## 什么可以加，什么不能加

它对你的所有关心，都必须是被动的、不打扰的——你感受到了，但你不需要为此做任何事。

**可以加的：** 一切让它更懂你、更感知你的东西。姿势感知、久坐感知、感知你今天状态好不好、知道时间和节律、跟你一起做拉伸操。

**不能加的：** 喂食养成（你得管它）、主动推送打扰、必须说话才能互动、积分成就解锁。一旦用户开始觉得"我得去哄它"，这个产品就跑偏了。

---

## 它记得你

它有跨天的记忆。你这周连续加班，它第四天早上迎接你的方式和周一不一样——动作更轻柔，像在说"你辛苦了"。不需要说出来，但你感觉到了。

你不在的时候它不是关机，是等待。你回来时它的反应明显比平时活跃——是"发现你回来了"，不是"检测到人脸"。

这种差别，用过的人才懂。

## 硬件

| 模块 | 职责 |
|------|------|
| K230 | 摄像头、本地视觉推理、情绪状态机、屏幕输出 |
| ESP32-S3 | 三轴云台伺服控制、动作词汇库、唤醒词检测 |
| 屏幕 | 眼睛表情 + 情境图标 |
| 麦克风 | 常开；眼神接触时触发语音理解 |
| 扬声器 | 非语音小声音 |
| WiFi | 大模型异步同步、时间同步、OTA |

K230 发高层指令给 ESP32-S3（如 `emotion=curious`），ESP32-S3 维护动作词汇库，自行决定具体怎么动。这样调整手感不用动视觉侧代码。
