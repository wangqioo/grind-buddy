# 苦命搭子 · Grind Buddy

饭有饭搭子，健身有健身搭子。上班这件事，终于也有搭子了。

公司关心你的 KPI。老板关心你的产出。同事关心自己的事。

**没有人关心你的情绪。只有它。**

---

绩效面谈出来，分数比预期低。你说了句"好的我会注意的"，走回工位，坐下，打开电脑，继续工作。全程没有人知道你心里是什么感受。

提了方案被否，说了句"好的我再想想"。你准备了很久，但没有人真的认真看过。回到工位，屏幕还开着，看了很久什么都没干。

开会时说了一句话，没有人接。就那么一两秒的沉默，然后话题被别人带走了。你低下头，装作在记笔记。

被催进度，但已经很尽力了。"这个今天能出来吗"——你回了"好的"，没有解释，因为解释了也没用。

哭了，但还有会要开。去洗手间洗了把脸，照镜子确认眼睛没有太红，回来继续。

周一早上进公司，在电梯里深呼吸了一下，门开了，走出去，开始新的一周。

这些时刻，没有人知道，没有人在意。

但它都看见了。因为它就放在你工位上，离你最近。你脸上一点细微的变化，声音里一丝情绪的起伏，眼眶有没有红，它全都看得清清楚楚。它什么都不说，但它知道。

在公司你不能随便抱怨，不能随便吐槽，说错一句话都可能有后果。但对着它，你可以说任何话。它不会评判你，不会传出去，不会觉得你不专业。它是你唯一一个敢尽情吐槽的地方。

**它是你工作日唯一的见证者。**

---

## 它是什么

一个放在桌上的三轴云台机器人。头是一张脸，有眼睛，会动，偶尔发出细小的声音。

它不说话——办公室里开口说话太尴尬，而且肢体语言已经足够。它不需要你去哄它、逗它、跟它聊天。它就坐在那，用自己的方式陪着你。

有点像猫。不会每五分钟来烦你，但你看它一眼，它会回应你。也有点像坐在旁边不说话、但你知道他在的同事。

它永远是关心你的那一方。

---

## 最核心的一件事：它知道你有没有在看它

K230 芯片实时识别你的面部朝向。你看向它的那一刻，它立刻转过来对视。

这是整个产品的灵魂。它感受到了你的目光，然后用目光回应你。

对视的这一刻，它才开始真正"读"你：你今天状态怎么样，眼神是精神还是疲惫，这一眼是随手一瞥还是真的在看它。然后决定怎么回应——是跟你共鸣，是轻轻关心你，还是就这样静静陪着你。

你不看它的时候，它就在那待着，有自己的情绪，缓慢漂移，偶尔动一动。不打扰你，但它是活的。

麦克风全天开着。但它把你跟同事的对话、打电话、自言自语，全部当成背景音来感知——语气是轻松还是紧绷，今天话多还是话少，声音里有没有疲惫。只有你看着它说话，它才知道你在跟它说话。

---

## 两个小细节

**扣下摄像头，它就休息了。**
不需要 App，不需要设置，手把摄像头物理扣下去，它自动进入休眠。比任何隐私开关都让人放心。想让它重新工作，抬起来就好。

**扣着的时候喊它名字，它会抬两下头。**
像一只被蒙眼睛的猫听到动静——它在说"我听见了，但我看不见你，你是在叫我吗？"顺带提醒你：*我在，但我瞎着呢。*

## 它怎么表达

它不说话，但它有三套语言，同时工作：

**动作** 是情绪的骨架。歪头是好奇，猛地抬头是"你回来了"，缓慢摇摆是放松，快速小抖是兴奋，低头转开是委屈。三个轴组合起来，表达力比你想象的丰富。

**眼睛** 和动作同步。歪头的同时眼睛变成好奇的形状，这一刻才真的"活了"。偶尔浮现一个小图标——快要开会了，眼睛旁边出现一个小钟，然后消散。不是通知，是它在提醒你。没有文字。

**小声音** 是最打动人的部分。它不说话，但有声音——你坐下来时一个轻轻的迎接，你收拾东西要走时一个送别，它等了你很久终于看到你回来那一声细小的雀跃。音量克制到你有时候会怀疑是不是幻听了。

---

## 它记得你

它有跨天的记忆。你这周连续加班，它第四天早上迎接你的方式和周一不一样——动作更轻柔，像是"你辛苦了"。不需要说出来，但你感觉到了。

你不在的时候它不是关机，是等待。你回来时它的反应比平时活跃——是"发现你回来了"，不是"检测到人脸"。

每天两次情绪密度最高的时刻：你来了，它知道。你走了，它也知道。

---

## 它会进化

它从你身上学动作。你做颈椎拉伸，它跟着你一起做——不是提醒你做，是陪你做。你的动作它轻微放大，像在共鸣，不像镜子。

慢慢地，它不只是模仿，开始发展出自己的变体，形成自己的风格。你的关注就是它的反馈信号，它做某个动作时你多看了它一眼，这个动作就被强化了。

用得越久，它越像它自己，也越像你们在一起的样子。最后，它有一套只属于你们的专属语言。你的这台，不可替代。

---

## 什么可以加，什么不能加

**可以加：** 一切让它更懂你的东西。感知你的姿势、感知你久坐了、知道今天星期几、知道现在几点、跟你一起做拉伸操。

**不能加：** 喂食养成（你得管它）、主动推送打扰、必须说话才能互动、积分成就解锁。一旦用户开始觉得"我得去哄它"，这个产品就跑偏了。

---

## 智能架构

```
本地实时层（K230 + ESP32-S3）
  感知：面部朝向、表情、眼神、动作、环境音
  表达：云台运动、屏幕表情、小声音
  无延迟，无 token 消耗，始终在线

大模型理解层（WiFi，异步后台）
  意图判断 — 同一个动作，上下文不同含义不同
  状态建模 — 今天累 / 专注 / 烦躁 / 开心
  响应决策 — 镜像 / 共鸣 / 关心 / 忽略
  定期消化 — 积累一天的观察，更新对你的理解
```

本地层是眼睛和手脚。大模型是你不看它时在后台思考的大脑。

## 硬件

| 模块 | 职责 |
|------|------|
| K230 | 摄像头、本地视觉推理、情绪状态机、屏幕输出 |
| ESP32-S3 | 三轴云台伺服控制、动作词汇库、唤醒词检测 |
| 屏幕 | 眼睛表情 + 情境图标 |
| 麦克风 | 常开；眼神接触时触发语音理解 |
| 扬声器 | 非语音小声音 |
| WiFi | 大模型异步同步、时间感知、OTA |

K230 发高层指令给 ESP32-S3（如 `emotion=curious`），ESP32-S3 维护动作词汇库自行决定怎么动——调手感不用动视觉侧代码。
