## 一、情绪状态机（K230）

### 状态定义

| 状态 | L1 | L2 | L3 |
|------|----|----|-----|
| WAITING | 刚离开不久 | 离开超过30分钟 | 今天还没来/消失一整天 |
| IDLE | 在但刚坐下 | 正常在场 | 在了很长时间没互动 |
| ATTENTIVE | 随手一瞥(<2s) | 正常对视(2-10s) | 长时间注视(>10s) |
| FOCUSED | 轻度专注 | 中度专注 | 深度专注 |
| TIRED | 有点累 | 明显疲惫 | 很疲惫/快撑不住 |
| HAPPY | 状态不错 | 明显开心 | 很开心 |
| LONELY | 有点想你 | 等了很久 | 今天一直没见到你 |

### 转移规则

```
任意状态
  └─ 检测到眼神接触 ──────────────▶ ATTENTIVE（最高优先级）

ATTENTIVE
  └─ 眼神离开 ──────────────────▶ 回到之前状态

人脸消失
  └─ 超时阈值 ──────────────────▶ WAITING（强度随时间递增）

人脸出现
  └─ ─────────────────────────▶ IDLE

IDLE
  ├─ 长时间静止不动 ────────────▶ FOCUSED
  ├─ 疲惫信号积累 ─────────────▶ TIRED
  └─ 正向情绪信号 ─────────────▶ HAPPY

WAITING(L3) / IDLE(L3)
  └─ ─────────────────────────▶ LONELY
```

### 强度计算

- **ATTENTIVE**：注视时长直接决定
- **TIRED / HAPPY**：视觉+音频信号累积评分，防止单帧误触发
- **FOCUSED**：静止时长 + 无眼神接触时长
- **WAITING / LONELY**：计时器

## 二、视觉模型流水线（K230）

四个任务串联执行，前一步是后一步的前提：

```
人脸检测（一直跑）
  └─ 检测到人脸
       └─ 人脸识别
            ├─ 陌生人 ──▶ CMD:action:alert
            └─ 主人
                 └─ 面部朝向识别
                      └─ 看着它（眼神接触）
                           └─ 表情/情绪识别
```

| 任务 | 触发时机 | 模型来源 |
|------|---------|---------|
| 人脸检测 | 一直跑 | K230 官方 SDK |
| 人脸识别 | 检测到人脸时 | K230 官方 SDK |
| 面部朝向识别 | 识别到主人时 | K230 官方头部姿态估计 |
| 表情/情绪识别 | 眼神接触时 | K230 官方表情分类模型 |

### 首次注册流程

```
1. 开机 → 屏幕引导注册动画
2. 用户打开小程序 → 蓝牙连接设备
3. 小程序发送注册指令 → ESP32-S3 → UART → K230
4. K230 采集人脸 → 生成特征向量 → 存储本地
5. 注册完成 → 小程序确认 → 进入正常工作模式
```

## 三、UART 指令集（K230 → ESP32-S3）

格式：`CMD:<type>:<name>:<intensity>`

```
# 情绪指令（持续性，状态机驱动）
CMD:emotion:waiting:1/2/3
CMD:emotion:idle:1/2/3
CMD:emotion:attentive:1/2/3
CMD:emotion:focused:1/2/3
CMD:emotion:tired:1/2/3
CMD:emotion:happy:1/2/3
CMD:emotion:lonely:1/2/3

# 动作指令（一次性触发）
CMD:action:greet          # 你来了
CMD:action:farewell       # 你走了
CMD:action:nod_twice      # 扣盖时听到唤醒词
CMD:action:stretch        # 跟你一起做颈椎操
CMD:action:cheer          # 雀跃庆祝
CMD:action:comfort        # 你揉眼睛，轻轻关心
CMD:action:alert          # 陌生人靠近

# 系统指令
CMD:system:sleep          # 摄像头扣下，休眠
CMD:system:wake           # 摄像头打开，唤醒
CMD:system:boot           # 开机动作
```

### 指令执行映射示例

```
CMD:emotion:tired:2  →  云台: 缓慢低头，轻微摇摆
                     →  屏幕: 眼皮半耷拉
                     →  声音: 低沉短促小声音
```

## 四、ESP32-S3 FreeRTOS 任务调度

### 任务划分

```
Task_UART      优先级: 高   接收 K230 指令，解析后广播到 Event Group
Task_Motion    优先级: 高   三轴伺服控制，执行动作曲线
Task_Screen    优先级: 中   屏幕表情渲染，与动作同步
Task_Sound     优先级: 中   声音播放
Task_WakeWord  优先级: 高   唤醒词检测，睡眠时也跑
Task_BLE       优先级: 低   蓝牙连接，处理小程序指令
```

### 同步机制

用 Event Group 实现一条指令同时触发三个任务：

```c
// Task_UART 解析指令后广播
xEventGroupSetBits(emotion_event, EMOTION_TIRED_L2);

// Task_Motion、Task_Screen、Task_Sound 同时监听同一 event
// 各自执行对应输出，天然同步
```

### 睡眠状态（摄像头扣下）

```
收到 CMD:system:sleep
  → Task_Motion   挂起
  → Task_Screen   挂起（息屏）
  → Task_Sound    挂起
  → Task_BLE      继续运行
  → Task_WakeWord 继续运行

Task_WakeWord 检测到唤醒词时
  → 不唤醒其他任务
  → 任务内部直接驱动伺服抬两下头
```

## 五、智谱 GLM 接入（异步，非实时）

**不在实时交互链路里**，避免网络延迟影响响应。

### 调用时机

| 场景 | 触发条件 | 作用 |
|------|---------|------|
| 眼神接触时 | 用户看着它说话 | 提取意图/情绪，更新状态机参数 |
| 每日定期 | 每天固定时间 | 消化今日积累数据，更新用户模型 |
| 状态模糊时 | 本地判断置信度低 | 辅助意图判断 |

### 数据流

```
K230 积累今日观察数据（本地存储）
  └─ 定期打包发送给智谱 GLM
       └─ 返回行为参数
            └─ 更新状态机权重 / 情绪基线
```
